# feat: Work tracker skill

Portable skill that manages work items in a project's `work/` directory. Bootstraps the directory structure on first use, creates/moves/scans items, and carries the conventions with it so they don't need to be copy-pasted into every project's CLAUDE.md.

## Current state

First draft of `skills/work-tracker/SKILL.md` is written. Covers:
- Bootstrap (creates `work/` structure if missing)
- Work item format (frontmatter with id, status, created, summary)
- File naming convention (`YYYYMMDD.<type>.<slug>.md`)
- Operations: create, move, scan/review
- Backlog management (grep-friendly format)
- Cross-referencing by id

## Changes

- **2026-03-25** — Added unsupervised ticket support: `backlog/unsupervised/` for queuing, `active/unsupervised/` for in-progress autonomous work. Items get `mode: unsupervised` frontmatter and must be self-contained with acceptance criteria. `active/` itself remains the human+model space.
- **2026-04-21** — Added session-bookends behaviour. The skill now ships `agents-snippet.md`, a delimited block (`<!-- BEGIN/END work-tracker:session-bookends -->`) projects paste into their `AGENTS.md` / `CLAUDE.md`. Tells the agent to proactively summarise last session at start (not wait for "where were we") and to remind the human to signal wrap-up so notes get captured. SKILL.md gained an "Installing the session-bookend behaviour" pointer. Block pasted into this repo's own CLAUDE.md. Rationale: skills can't ship hooks (harness-specific), and relying on the human to remember phrases is fragile — AGENTS.md is always loaded, so proactive prompting travels with the skill.
