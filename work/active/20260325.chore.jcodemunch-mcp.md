## What we know

**Connection**: working via `.mcp.json`, 3 repos already indexed (@oneput, jsed, workers-sdk).

**Tool surface** (25 tools, grouped):

### Indexing
- `index_repo` — index a GitHub repo (fetches, parses ASTs, extracts symbols)
- `index_folder` — index a local folder
- `index_file` — surgical single-file reindex within an existing index
- `invalidate_cache` — delete index, force full reindex
- `wait_for_fresh` — wait for watcher reindex to complete

### Discovery / orientation
- `list_repos` — all indexed repos
- `resolve_repo` — path → repo identifier (O(1))
- `get_repo_outline` — high-level: dirs, file counts, languages, symbol counts
- `suggest_queries` — entry-point files, top keywords, example queries
- `get_file_tree` — file tree, optionally filtered by path prefix

### Symbol search & inspection
- `search_symbols` — query across repo, returns signatures + summaries. Has `detail_level` (compact/standard/full) and `token_budget` mode
- `get_file_outline` — all symbols in a file (batch: `file_paths`)
- `get_symbol_source` — full source of one or many symbols (batch: `symbol_ids`)
- `get_context_bundle` — source + imports for symbol(s), deduped. Markdown output option
- `get_related_symbols` — heuristic clustering: co-location, shared importers, name overlap

### Text search
- `search_text` — full-text across file contents, regex support, context lines
- `search_columns` — column metadata search (dbt/SQLMesh/database catalogs)

### Dependency & impact analysis
- `get_dependency_graph` — file-level import graph, up to 3 hops, bidirectional
- `find_importers` — what files import a given file (batch: `file_paths`)
- `find_references` — where is an identifier used (batch: `identifiers`)
- `check_references` — is an identifier referenced anywhere (dead-code detection, batch)
- `get_blast_radius` — all files affected by changing a symbol
- `get_class_hierarchy` — inheritance tree (ancestors + descendants)

### Diffing
- `get_symbol_diff` — diff symbol sets between two indexed snapshots (branch comparison)
