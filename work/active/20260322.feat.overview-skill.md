# feat: Overview skill

A routing skill that gives the human (and agent) a grab handle on the big picture. Covers multiple concerns via sub-skills, loaded on demand.

## Sub-skills

- **Themes** — what is this project about, what are the driving ideas. For this repo: the four themes in `docs/`. For other projects: whatever docs/ captures about purpose and direction.
- **Vocabulary** — foundational layer, applies to everything. Not tied to architecture or any specific system. The compression mechanism — shared terms that let humans and agents communicate at high bandwidth. Bridge from high-level thinking to low-level precision. Helps the human parse their own ideas clearly. The agent uses it to compress output into terms the human already holds.
- **Architecture narrative** — situational, only fires when there's a system to explain. Tells the story of how a system fits together. One consumer of the shared vocabulary, not the reason vocabulary exists.
- **Session resume** — where were we, what's in flight, offered prompts (copy-pasteable prompts the human can fire).

## Memory layers and skill boundaries

Three concerns, each with its own skill/mechanism:

| Concern | Layer | Skill | Covers |
|---------|-------|-------|--------|
| **Vocabulary** | Ambient — always on | Shared language (docs/vocabulary.md) | Compression substrate. Enables clear thinking and high-bandwidth communication at every layer. Not tied to any specific system or timescale. |
| **Overview** | Long-term — the "why", strategic | Overview skill (this ticket) | Themes, architecture narrative, offered prompts, doorways into things, paths. Conveys purpose, direction, and orientation. Lives in `docs/`. |
| **Work-tracker** | Mid-to-short-term — the "what", tactical | Work-tracker skill | Work items (days/weeks+), session logs (per session), conversation context (ephemeral). Manages what's in flight, what we did, where we left off. Lives in `work/` and `.sessions/`. |

Overview and work-tracker aren't competing — they're different layers operating on the same project. Overview says "here's why this matters and where it's going." Work-tracker says "here's what's in flight and where you left off." Vocabulary is the substrate both sit on.

## Routing pattern

Uses the streamlit-style routing pattern (see `/Users/danb/work/FPP_AI/.claude/skills/developing-with-streamlit/`): top-level SKILL.md triggers broadly, then loads appropriate sub-skill based on context. New to the project? Themes + vocabulary. Returning? Session resume + work overview. Working on a system? Architecture narrative.
